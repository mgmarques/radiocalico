# Internationalization (i18n)

## Supported Languages

3 languages: English (default), Brazilian Portuguese (pt-BR), Spanish (es).

## Core Rules

| # | Rule |
|---|------|
| I-1 | **NEVER translate song metadata** — artist names, track titles, album names must always remain in their original language |
| I-2 | UI labels use `data-i18n` attributes + `t(key)` helper function |
| I-3 | `_TRANSLATIONS` object in `player.js` maps keys to `{en, pt, es}` |
| I-4 | `applyLanguage()` updates all DOM elements with `data-i18n`, re-renders history, re-fetches active AI content |
| I-5 | Quiz button label is ALWAYS "Quiz" — no `data-i18n` attribute on the Quiz button |
| I-6 | Language persisted in `localStorage` key `rc-lang` |
| I-7 | LLM system prompts MUST include "NEVER translate song titles, album names, or artist names" |

## Implementation

- `t(key)` reads `_currentLang` (default `'en'`) and returns `_TRANSLATIONS[key][_currentLang]` or falls back to English
- `applyLanguage(lang)` sets `_currentLang`, updates `localStorage` `rc-lang`, iterates `[data-i18n]` elements, re-renders history, re-fetches active LLM panel content
- Language selector: radio buttons in the settings dropdown (same pattern as theme/quality selectors)

## LLM Integration

- `_LANG_TO_LLM` maps language codes to LLM prompt language names: `{en: 'English', pt: 'Brazilian Portuguese', es: 'Spanish'}`
- Song info and quiz prompts append language instruction to system prompt
- System prompts MUST instruct: "Respond in {language}. NEVER translate song titles, album names, or artist names — keep them in their original language."

## Adding a New Language

When adding a new language:

1. Add translation object to `_TRANSLATIONS` in `player.js` (copy from `en`, translate all values)
2. Add entry to `_LANG_TO_LLM` mapping in `player.js`
3. Add radio button in `static/index.html` settings section (language group)
4. Add test cases in `player.test.js` for the new language
5. Update this file's language count
6. Update `CLAUDE.md` and `README.md` language lists

## Key Files

- `static/js/player.js` — `_TRANSLATIONS`, `_LANG_TO_LLM`, `t()`, `applyLanguage()`
- `static/index.html` — `data-i18n` attributes on UI labels, language radio buttons in settings
- `api/llm_service.py` — language parameter in LLM system prompts
- `static/js/player.test.js` — i18n test cases
